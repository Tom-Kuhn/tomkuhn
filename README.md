# tomkuhn.co.uk

Personal website for Tom Kuhn — Staff Engineer, Architect, and Problem Solver.

## Technologies

- **[Jekyll](https://jekyllrb.com/) 4.4** — static site generator
- **Ruby** — runtime (3.3 recommended)
- **Liquid** — templating language used in layouts and includes
- **Sass/SCSS** — styles compiled by Jekyll
- **GitHub Actions** — CI/CD pipeline deploying to GitHub Pages on push to `main`

## Plugins

| Plugin | Purpose |
|---|---|
| [jekyll-feed](https://github.com/jekyll/jekyll-feed) | Auto-generates an Atom feed at `/feed.xml` |
| [jekyll-seo-tag](https://github.com/jekyll/jekyll-seo-tag) | Injects SEO meta tags (`<title>`, Open Graph, Twitter Card, canonical URL) via `{% seo %}` in `_includes/head.html` |
| [jekyll-assets](https://github.com/envygeeks/jekyll-assets) | Asset pipeline with fingerprinting and compression. Sources configured from `assets/css`, `assets/js`, and `assets/images`. Use `{% asset filename %}` in templates to reference processed assets |
| [jekyll-redirect-from](https://github.com/jekyll/jekyll-redirect-from) | Add `redirect_from: /old-url/` to any page or post front matter to create redirect pages |
| [jekyll-tagging](https://github.com/pattex/jekyll-tagging) | Provides `{% tag_cloud %}` and `{% tags_list %}` Liquid tags for rendering tag clouds and lists |
| [jekyll-archives](https://github.com/jekyll/jekyll-archives) | Auto-generates archive pages by year (`/archive/:year/`), month (`/archive/:year/:month/`), tag (`/tag/:name/`), and category (`/category/:name/`) |
| [jekyll-mentions](https://github.com/jekyll/jekyll-mentions) | Converts `@username` mentions in post content into GitHub profile links |
| [jekyll-gist](https://github.com/jekyll/jekyll-gist) | Embed GitHub Gists in posts using `{% gist gist-id %}` or `{% gist username/gist-id %}` |
| [github-metadata](https://github.com/jekyll/github-metadata) | Exposes GitHub repository metadata via `site.github` (e.g. `site.github.repository_name`) |

## Prerequisites

- Ruby (3.3 recommended)
- Bundler (`gem install bundler`)

## Getting Started

Install dependencies:

```bash
bundle install
```

Serve locally with live reload:

```bash
bundle exec jekyll serve
```

The site will be available at `http://localhost:4000`.

Build for production:

```bash
bundle exec jekyll build
```

Output is written to `_site/`.

## Using the plugins

### Adding tags to a post

Add a `tags` list to any post's front matter:

```yaml
---
layout: post
title: "My post title"
date: 2026-01-01
tags: [architecture, engineering, career]
---
```

Tag archive pages are automatically generated at `/tag/:name/`. A tag cloud can be rendered anywhere using `{% tag_cloud %}`.

### Redirecting old URLs

Add `redirect_from` to any page or post front matter:

```yaml
---
redirect_from:
  - /old-path/
  - /another/old/path/
---
```

### Mentioning GitHub users

Write `@username` anywhere in post content — jekyll-mentions will automatically link it to `https://github.com/username`.

### Embedding a Gist

```liquid
{% gist 5555251 %}
{% gist parkr/931c1c8d465a04042403 %}
```

### Using GitHub repository metadata

Access repository info in templates:

```liquid
{{ site.github.repository_name }}
{{ site.github.owner_name }}
```

> **Note:** `github-metadata` requires a `JEKYLL_GITHUB_TOKEN` environment variable when running locally. On GitHub Actions the token is available automatically.

## Deployment

Pushing to the `main` branch triggers the GitHub Actions workflow at [.github/workflows/deploy.yml](.github/workflows/deploy.yml), which builds the site and deploys it to GitHub Pages automatically.
